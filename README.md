# 8-2-journal
## CS 230 Module Eight Journal

### Client Summary
The client was The Gaming Room, a company that wanted to expand their game into a web-based application that could support multiple teams and players. The goal was to design a scalable, distributed system that ensures only one instance of the game runs while allowing many users to access it across different platforms.

### What I Did Well
I did well in clearly organizing the system design and explaining my choices. I effectively compared different platforms and technologies, which helped justify why certain solutions were better for the client’s needs.

### Helpful Design Process
Working through the design document helped me plan the system before coding. It allowed me to think about scalability, constraints, and compatibility early, which made development more efficient and reduced potential issues.

### What I Would Improve
If I could revise one part, I would improve the comparison of different platforms by adding more detail and real-world examples. This would make my reasoning stronger and more professional.

### Interpreting User Needs
I focused on the client’s need for scalability, performance, and cross-platform support. I designed the system to handle multiple users while maintaining a single game instance. Considering user needs is important because it ensures the final product is useful and meets expectations.

I approached the design by analyzing requirements first, then evaluating different solutions
